## Overview

The overview endpoint provides a complete snapshot of your agent's performance metrics. It returns summary statistics, timeseries data for tokens/costs/messages, cost breakdown by model, recent activity, and active skills.

This endpoint intelligently switches between hourly and daily aggregations based on the selected time range, providing optimal granularity for visualization.

## Authentication

Requires session authentication (cookie) or API key via `X-API-Key` header.

## Query Parameters

<ParamField query="range" type="string" default="24h">
  Time range for analytics. Valid values: `1h`, `6h`, `24h`, `7d`, `30d`
  
  - Hourly data returned for: `1h`, `6h`, `24h`
  - Daily data returned for: `7d`, `30d`
</ParamField>

<ParamField query="agent_name" type="string">
  Filter analytics to a specific agent. If omitted, returns data for all agents.
</ParamField>

<ResponseField name="active_skills" type="array">
  Skills that were executed in the time range, ordered by run count
  
  <Expandable title="active_skills item fields">
    <ResponseField name="name" type="string">
      Skill name
    </ResponseField>

      Agent that executed this skill

    <ResponseField name="run_count" type="number">
      Number of executions in the time range
    </ResponseField>
    
    <ResponseField name="last_active_at" type="string">
      ISO 8601 timestamp of last execution
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Always `active` for this endpoint
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="has_data" type="boolean">
  Whether any telemetry data exists for this user/agent
</ResponseField>

## Example Request

```bash cURL
curl -X GET "https://api.manifest.build/api/v1/overview?range=24h&agent_name=my-agent" \
  -H "X-API-Key: your-api-key"
```

## Notes

- **Trend calculations**: Compares current period vs. previous period of equal length. For example, `24h` range compares last 24 hours vs. previous 24 hours.
- **Hourly vs. Daily**: Ranges `1h`, `6h`, `24h` return hourly timeseries. Ranges `7d`, `30d` return daily timeseries.
- **Caching**: Responses are cached for 5 minutes (see `DASHBOARD_CACHE_TTL_MS`).
- **Multi-tenancy**: Data is automatically filtered by authenticated user. Users only see their own agents' data.
- **Cost sanitization**: Negative costs (invalid pricing data) are treated as 0.
